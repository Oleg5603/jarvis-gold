# ТЗ — Мультивендорный маркетплейс на WooCommerce

**Стек:** WordPress + WooCommerce · PHP 8 · Custom REST API · React SPA · MySQL + Redis · Action Scheduler · aiogram

---

## Функциональные требования

### Кабинет продавца
- Регистрация и модерация (загрузка документов, статус проверки)
- Многоэтапный онбординг: шаги регистрации → загрузка документов → ожидание модерации → активация
- Личный каталог товаров, управление остатками и ценами
- История заказов и статистика продаж
- Публичная страница продавца с рейтингом и отзывами
- Мобильная адаптация кабинета (breakpoints 768/480px)

### Логика маркетплейса
- Автоматическое разделение заказа на суб-заказы по продавцам
- Настраиваемая комиссия (фиксированная / процентная, индивидуально по продавцу или категории)
- Очередь модерации новых товаров перед публикацией (WP-Cron / Action Scheduler)
- Автоматическая проверка дубликатов товаров при загрузке
- Логика возвратов: инициация покупателем → подтверждение продавцом → возврат средств

### Выплаты
- Автоматические выплаты: **Stripe Connect** (приоритет) или PayPal Payouts
- Ручной экспорт реестра для бухгалтерии (CSV/Excel) как резервный режим
- График выплат: еженедельно или по достижению порога суммы

### Рейтинги и отзывы
- Лимит одного отзыва на заказ (защита от накрутки)
- Фильтрация подозрительной активности (rate limiting по IP + аккаунту)

### Уведомления
- Email + Telegram (aiogram-бот) при новом заказе, изменении статуса, запросе на выплату

### Административная панель
- Дашборд: общая выручка, комиссии, активные продавцы, топ товаров
- Управление модерацией продавцов и товаров
- Встроенный чат поддержки (для продавцов)

### Для роста (опционально)
- Трекинг заказов для покупателей
- PWA или мобильное приложение
- Мультивалютность

## Безопасность и производительность API

### JWT-аутентификация для React SPA
- Авторизация через `/wp-json/jwt-auth/v1/token`
- Access token (TTL 15 мин) + Refresh token в httpOnly cookie
- Все `/wp-json/marketplace/v1/*` защищены middleware
- Отзыв токенов при деактивации продавца

### Кеширование (Redis)
- Каталог продавца: TTL 5 мин
- Статистика продаж: TTL 15 мин
- Публичные рейтинги: TTL 30 мин
- Инвалидация по событиям: `woocommerce_order_status_changed`, `save_post_{product}`, `wp_insert_comment`
- Заголовки `Cache-Control: private` на авторизованных эндпоинтах

### Защита
- Nonce на всех формах WordPress
- Capability checks (`current_user_can`) на каждом REST-эндпоинте
- reCAPTCHA v3 на регистрации и входе
- Подготовленные запросы (`$wpdb->prepare`) — без сырого SQL
- Rate limiting на API (429 при превышении лимита)

### База данных
- Индексы: `vendor_id`, `order_id`, `product_id`, `created_at` во всех кастомных таблицах
- Отдельная таблица `mp_vendor_stats` для агрегированной статистики (не COUNT на лету)
- Партиционирование таблицы заказов при > 1M записей

## Инфраструктура

- **Hosting:** Managed WordPress (WP Engine / Kinsta / Cloudways) — обязательно
- **Redis:** включён на уровне хостинга или отдельный инстанс
- **Бэкапы:** ежедневные, хранение 30 дней
- **CDN:** Cloudflare (бесплатный план на старте)
